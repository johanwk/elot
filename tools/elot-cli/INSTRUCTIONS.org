#+DESCRIPTION: Guidance for LLM-supported development of the Org->OMN pipeline

* Project Context
=elot-cli= is a TypeScript port of the [[https://github.com/johanwk/elot][ELOT]] Emacs Lisp pipeline. It converts Org-mode outlines into OWL Manchester Syntax (OMN). It uses the =orga= parser (v4+) which is unified-based.

* Current Status (Milestone 5)
Core logic for parsing headlines, properties, and generating OMN frames is implemented. However, a few "plumbing" bugs prevent perfect output, particularly regarding prefix handling and taxonomy.

* Key File Roles
- =src/types.ts=: Core interfaces (=ElotNode=, =DescriptionItem=).
- =src/parseHierarchy.ts=: The main AST-to-tree converter. Handles property extraction and level-based nesting.
- =src/entityFromHeader.ts=: Regex-heavy parser for headline titles (URIs/CURIEs).
- =src/annotationValue.ts=: Classifies and formats values (Strings, URIs, URNs).
- =src/omnFrame.ts=: Formats individual =Class:=, =ObjectProperty:= frames.
- =src/generateOmn.ts=: Assembles the final document.
