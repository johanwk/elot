* Introduction

This file defines =elot-mode=, a minor mode that activates when editing
an ELOT ontology document in Org-mode.  It replaces the old
=elot-defaults.el= approach, where every ELOT file required a first-line
file-local variable:

#+begin_example
# -*- eval: (load-library "elot-defaults") -*-
#+end_example

That mechanism had several problems:

- Every file open triggered a security prompt ("Allow eval in local
  variables?")
- The setup was an imperative bag of side-effects with no teardown path
- Hooks were added locally but never removed
- Keybindings were set via =local-set-key= with no proper keymap

A =define-minor-mode= solves all of these: it provides a toggle, a
dedicated keymap, a modeline lighter, and clean enable/disable
functions.

** Architecture

#+begin_example
elot-mode.org          → elot-mode.el          (this file: minor mode definition)
elot-tangling.org      → elot-tangle.el        (AST parsing, OMN generation)
elot-label-display.org → elot-label-display.el  (fontification, overlays, lookup)
elot-defs.org          → elot.el                (legacy: gradually being emptied)
#+end_example

=elot-mode.el= is thin orchestration glue.  It =require='s the modules,
defines the minor mode, and wires up hooks and keybindings.

** Activation

Users can activate =elot-mode= in several ways:

1. File-local variable (replaces the old =eval:= cookie):
   : # -*- eval: (elot-mode 1) -*-

2. A =.dir-locals.el= in the project root:
   : ((org-mode . ((eval . (elot-mode 1)))))

3. Automatic detection on =org-mode-hook= (see [[*Auto-detection][Auto-detection]]).

** Buffer-local settings
:PROPERTIES:
:CUSTOM_ID: buffer-local-settings
:END:

These are the non-LaTeX settings formerly set inside the =progn= block of
=elot-defaults.el=.  They are applied when the mode is enabled and have
no teardown needed (they are buffer-local and vanish with the buffer).

#+name: src-mode-buffer-settings
#+begin_src emacs-lisp
  (defun elot-mode--set-buffer-locals ()
    "Set buffer-local variables for an ELOT buffer.
  These correspond to the `setq-local' block that was in `elot-defaults.el',
  minus the LaTeX-specific settings."
    (setq-local
     org-confirm-babel-evaluate nil
     org-export-allow-bind-keywords t
     org-babel-default-inline-header-args '((:exports . "code"))
     tempo-interactive t
     time-stamp-line-limit 100
     time-stamp-format "%Y-%m-%d %H:%M"
     time-stamp-active t
     time-stamp-start "(version of "
     time-stamp-end ")"
     org-startup-folded 'show2levels
     org-export-with-sub-superscripts nil     ; preserve "_" in exports
     org-export-headline-levels 8             ; deep numbering
     org-export-with-section-numbers 8))      ; deep numbering
#+end_src

** Syntax table
:PROPERTIES:
:CUSTOM_ID: syntax-table
:END:

ELOT treats =:= and =_= as word constituents so that CURIEs like
=ex:MyClass= and identifiers like =BFO_0000001= are recognised as
single words by movement commands and =thing-at-point=.

We derive a syntax table from =org-mode-syntax-table= so we can swap it
in on enable and restore the original on disable.

#+name: src-mode-syntax-table
#+begin_src emacs-lisp
  (defvar elot-mode-syntax-table
    (let ((st (make-syntax-table org-mode-syntax-table)))
      (modify-syntax-entry ?: "w" st)
      (modify-syntax-entry ?_ "w" st)
      st)
    "Syntax table for `elot-mode'.
  Treats `:' and `_' as word-constituent characters.")
#+end_src

** Hook management
:PROPERTIES:
:CUSTOM_ID: hook-management
:END:

All buffer-local hooks are added on enable and removed on disable.
This is the key improvement over =elot-defaults.el=, which only added
hooks and never removed them.

#+name: src-mode-hooks
#+begin_src emacs-lisp
  (defvar-local elot-mode--idle-timer nil
    "Handle for the idle timer that sets up label-display.")

  (defun elot-mode--add-hooks ()
    "Add buffer-local hooks for an ELOT buffer."
    ;; Pre-tangle: remember source file and generate OMN from outline
    (add-hook 'org-babel-pre-tangle-hook #'elot--remember-org-source nil t)
    (add-hook 'org-babel-pre-tangle-hook #'elot-tangle-buffer-to-omn nil t)
    ;; Post-tangle: convert OMN to Turtle via ROBOT
    (add-hook 'org-babel-post-tangle-hook #'elot-tangled-omn-to-ttl nil t)
    ;; After babel execution: refresh inline images
    (add-hook 'org-babel-after-execute-hook #'org-redisplay-inline-images nil t)
    ;; After save: refresh link abbreviations and slurp data
    (add-hook 'after-save-hook #'elot-update-link-abbrev nil t)
    (add-hook 'after-save-hook #'elot-slurp-to-vars nil t))

  (defun elot-mode--remove-hooks ()
    "Remove buffer-local hooks added by `elot-mode--add-hooks'."
    (remove-hook 'org-babel-pre-tangle-hook #'elot--remember-org-source t)
    (remove-hook 'org-babel-pre-tangle-hook #'elot-tangle-buffer-to-omn t)
    (remove-hook 'org-babel-post-tangle-hook #'elot-tangled-omn-to-ttl t)
    (remove-hook 'org-babel-after-execute-hook #'org-redisplay-inline-images t)
    (remove-hook 'after-save-hook #'elot-update-link-abbrev t)
    (remove-hook 'after-save-hook #'elot-slurp-to-vars t))
#+end_src

** Enable and disable
:PROPERTIES:
:CUSTOM_ID: enable-disable
:END:

The enable function does what the =progn= block in =elot-defaults.el= used
to do, but in a structured and reversible way.

#+name: src-mode-enable-disable
#+begin_src emacs-lisp
  (defun elot-mode--enable ()
    "Set up ELOT in the current Org buffer."
    ;; 1. Buffer-local variables
    (elot-mode--set-buffer-locals)

    ;; 2. Syntax table
    (set-syntax-table elot-mode-syntax-table)

    ;; 3. Ingest the Library of Babel
    (let ((lob-file (concat (file-name-directory
                             (locate-library "elot-tangle"))
                            "elot-lob.org")))
      (when (file-exists-p lob-file)
        (org-babel-lob-ingest lob-file)))

    ;; 4. Refresh link abbreviations from prefix table
    (elot-update-link-abbrev)

    ;; 5. Hooks
    (elot-mode--add-hooks)

    ;; 6. Label-display: start after a short idle delay
    (setq elot-mode--idle-timer
          (run-with-idle-timer 1.0 nil #'elot-label-display-setup))

    ;; 7. Set initial fold visibility
    (org-cycle-set-startup-visibility))

  (defun elot-mode--disable ()
    "Tear down ELOT in the current Org buffer."
    ;; 1. Remove hooks
    (elot-mode--remove-hooks)

    ;; 2. Cancel idle timer
    (when (timerp elot-mode--idle-timer)
      (cancel-timer elot-mode--idle-timer)
      (setq elot-mode--idle-timer nil))

    ;; 3. Remove label-display overlays
    (when (fboundp 'elot-remove-prop-display)
      (with-silent-modifications
        (elot-remove-prop-display)))

    ;; 4. Restore syntax table
    (set-syntax-table org-mode-syntax-table))
#+end_src

** The minor mode definition
:PROPERTIES:
:CUSTOM_ID: minor-mode-definition
:END:

This is the central =define-minor-mode= form.

#+name: src-mode-define
#+begin_src emacs-lisp
  ;;;###autoload
  (define-minor-mode elot-mode
    "Minor mode for authoring OWL ontologies in Org-mode with ELOT.

  When enabled, sets up:
  - Tangling hooks for OMN export and ROBOT conversion
  - Label-display overlays (toggle with \\[elot-toggle-label-display])
  - Hydra menu (\\[elot-hydra/body]) with templates and export commands
  - Syntax table tweaks for CURIE navigation
  - Library of Babel ingestion

  When disabled, all hooks, timers, and overlays are cleanly removed."
    :lighter " ELOT"
    :keymap elot-mode-map
    (if elot-mode
        (elot-mode--enable)
      (elot-mode--disable)))
#+end_src

** Auto-detection
:PROPERTIES:
:CUSTOM_ID: auto-detection
:END:

Optionally, ELOT mode can be activated automatically when an Org buffer
contains the property =:ELOT-context-type: ontology=.  This eliminates
the need for any file-local variable at all.

The detection function is added to =org-mode-hook=.  It searches for the
ELOT marker property in the first few thousand characters of the
buffer — fast enough for interactive use.

#+name: src-mode-auto-detect
#+begin_src emacs-lisp
  (defcustom elot-mode-auto-detect t
    "When non-nil, automatically enable `elot-mode' in ELOT buffers.
  Detection looks for `:ELOT-context-type: ontology' in the buffer."
    :type 'boolean
    :group 'elot)

  (defun elot-mode--maybe-enable ()
    "Enable `elot-mode' if the current Org buffer looks like an ELOT file."
    (when (and elot-mode-auto-detect
               (derived-mode-p 'org-mode)
               (not (bound-and-true-p elot-mode))
               (save-excursion
                 (goto-char (point-min))
                 (re-search-forward
                  ":ELOT-context-type:.*ontology" nil t)))
      (elot-mode 1)))

  (add-hook 'org-mode-hook #'elot-mode--maybe-enable)
#+end_src

** Backward compatibility shim
:PROPERTIES:
:CUSTOM_ID: backward-compat
:END:

Existing ELOT files that use the old =eval:= cookie can be migrated
gradually.  During the transition, =elot-defaults.el= can be reduced to:

#+begin_example
(require 'elot-mode)
(elot-mode 1)
#+end_example

Or users can replace:
: # -*- eval: (load-library "elot-defaults") -*-
with:
: # -*- eval: (elot-mode 1) -*-

Once auto-detection is relied upon, even that line can be removed.

** Provide the library

#+name: src-mode-provide
#+begin_src emacs-lisp :comments nil
  (provide 'elot-mode)
  ;;; elot-mode.el ends here
#+end_src

* Appendix: What was removed from =elot-defaults.el=

For reference, here are the LaTeX-specific settings that were in
=elot-defaults.el= and are *deliberately excluded* from =elot-mode.el=
for now.  They need a complete rework before being re-included:

- =org-latex-src-block-backend=
- =org-latex-prefer-user-labels=
- =org-latex-image-default-scale=
- =org-latex-default-class=
- =org-latex-packages-alist= (extensive KOMA-Script configuration)
- =org-latex-classes= (the ="elot-scrreprt"= class definition)
- =elot-ontology-resource-section= (LaTeX section-level function)

These will be addressed in a future =elot-latex.org= module.
