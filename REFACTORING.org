* Overview

This document is the working project plan for refactoring the
tangle-to-OWL Manchester Notation (OMN) pipeline in ELOT.

The central ELOT capability is converting Org-mode outlines and
description lists into OWL Manchester Syntax.  The relevant code
currently lives scattered across =elot-defs.org= (~118 KB).  The goal
of this multi-phase effort is to:

- Extract the tangle-to-OMN functions into a dedicated =elot-tangle.org=
- Add ERT regression tests so refactoring does not break behaviour
- Improve docstrings and narrative documentation
- Refactor Org-reading functions to use the =org-element= API
- Harden OMN generation and taxonomy functions

*Note:* =elot-package/elot.el= is entirely tangled from =elot-defs.org=
and must never be edited directly.  All source changes go through =.org=
files.

*Note:* This =REFACTORING.org= file lives only on the refactoring branch
and should be removed before squash-merging to =main=.

* Data Flow

The pipeline converts Org structure into OMN output in two parallel
streams that are then combined.

#+BEGIN_EXAMPLE
Org outline headings  →  elot-org-list-siblings          →  hierarchy tree (nested lists)
Org description lists →  elot-org-descriptions-in-section →  annotation/restriction pairs
                                  ↓                                       ↓
                  elot-resource-taxonomy-from-header    elot-resource-declarations-from-header
                                  ↓                                       ↓
                  elot-resource-taxonomy-from-l         elot-resource-declarations
                                  ↓                                       ↓
                          OMN SubClassOf/SubPropertyOf        OMN Class: + annotations
#+END_EXAMPLE

* Function Reference

| Function                              | Purpose                                    | Source block          |
|---------------------------------------+--------------------------------------------+-----------------------|
| elot-org-list-siblings                | Reads Org outline into nested list         | src-heading-to-list   |
| elot-org-descriptions-in-section      | Reads description lists into pairs         | src-desc-lists        |
| elot-org-subsection-descriptions      | Maps headings → description lists          | src-desc-lists        |
| elot-entity-from-header               | Extracts OWL entity IRI from heading       | src-heading-to-list   |
| elot-omn-declare                      | Generates Class: declaration               | src-resource-declare  |
| elot-annotation-entries               | Formats annotation pairs as OMN            | src-resource-declare  |
| elot-restriction-entries              | Formats restriction axioms as OMN          | src-resource-declare  |
| elot-resource-declarations            | Combines declare + annotate + restrict     | src-resource-declare  |
| elot-resource-declarations-from-header | Top-level: Org ID → OMN declarations      | src-resource-declare  |
| elot-resource-taxonomy-from-l         | Recursive: list → SubClassOf axioms        | src-write-taxonomy    |

| elot-misc-axioms                      | DisjointClasses, EquivalentClasses etc.    | src-resource-declare  |

* Source Blocks to Extract (Phase 1)

The following named source blocks from =elot-defs.org= should be moved
into the new file =elot-tangle.org=:

- =src-heading-to-list= — outline traversal, entity extraction
  (includes =elot-org-list-siblings=, =elot-entity-from-header=)
- =src-desc-lists= — description list parsing
  (includes =elot-org-descriptions-in-section=, =elot-org-subsection-descriptions=)
- =src-resource-declare= — OMN declarations, annotations, restrictions
  (includes =elot-omn-declare=, =elot-annotation-entries=,
  =elot-restriction-entries=, =elot-resource-declarations=,
  =elot-resource-declarations-from-header=, =elot-misc-axioms=)
- =src-write-class= — oneof/disjoint helpers
  (includes =elot-class-oneof-from-header=, =elot-class-disjoint-from-header=)
- =src-write-taxonomy= — taxonomy generation
  (includes =elot-resource-taxonomy-from-l=, =elot-resource-taxonomy-from-header=)
- OMN keyword variables:
  =elot-omn-property-keywords=, =elot-omn-misc-keywords=, =elot-omn-all-keywords=
- =src-prefix-links= — prefix/link-abbrev handling
  (borderline; used by tangle pipeline but also by display functions)

* Design Decisions

This section captures design notes and rationale as they arise during
the work.

** Borderline code: =src-prefix-links=

The =src-prefix-links= block is used both by the tangle pipeline (to
resolve prefixed URIs into expanded forms for OMN output) and by the
interactive display functions (e.g., link-abbrev setup in Org buffers).

Decision pending: either move it and add a cross-reference, or duplicate
a minimal version for =elot-tangle.org= while keeping the original in
=elot-defs.org=.

** =org-link-abbrev-alist-local= dependency

Several tangle functions (notably =elot-annotation-entries= and
=elot-unprefix-uri=) rely on the buffer-local variable
=org-link-abbrev-alist-local= to filter and expand prefixed URIs.  This
tight coupling to buffer state complicates unit testing.

Phase 4 should consider making the prefix alist an explicit function
parameter rather than an implicit buffer-local variable.
