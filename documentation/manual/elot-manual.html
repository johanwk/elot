ELOT lets you author OWL ontologies in Org-mode, exporting to OWL Manchester Syntax, HTML and PDF documentation, and running SPARQL, ROBOT and rdfpuml against them from inside Emacs.

1. Preface

ELOT — the Emacs Literate Ontology Tool — is a package for authoring OWL ontologies in Org-mode. A single Org file is both the ontology source (tangled to OWL Manchester Syntax) and the published documentation (exported to HTML, PDF, or Info).

1.1. Who this manual is for

  • Ontology authors who want their ontology and its documentation to stay in sync, automatically.
  • Emacs users curious about how literate programming applies to knowledge engineering.
  • Newcomers to OWL who want a working ontology in front of them before reading a 400-page semantic-web textbook.

No prior experience with OWL or Org-mode is assumed for Part I. Parts II–IV assume you have completed the tutorial.

1.2. How to read this manual

This manual follows the Diátaxis structure:

  • Read Part I front-to-back the first time.
  • Dip into Part II when you have a specific task.
  • Consult Part III for lookups.
  • Read Part IV when you want to know why.

1.3. Conventions

  • code face for CURIEs, IRIs, function names, file paths.
  • ~C-c C-x r~ for keystrokes.
  • italic for new terms on first use.
  • M-x command-name for interactive commands.

1.4. Getting help

  • Project home: https://github.com/johanwk/elot
  • Issues and discussion: the GitHub tracker.
  • The ELOT menu in any elot-mode buffer lists the main commands.

2. Part I — Tutorial

This part is linear. Read it in order; it walks a first-time user through installing ELOT and building a small ontology end-to-end.

2.1. Installation

The fastest path is via MELPA once ELOT is accepted there. Until then, install from Git.

2.1.1. Prerequisites

  • Emacs 30.1 or later. ELOT uses peg.el, which became a built-in library in 30.1.
  • Java 11 or later, required by ROBOT.
  • ROBOT, an OWL command-line tool. Download robot.jar from the ROBOT releases page and note its path.
  • PlantUML (optional), for diagram rendering via rdfpuml.
  • Protégé 5.6 (optional), for cross-checking ontologies in a GUI.

2.1.2. Install ELOT from MELPA

When ELOT is on MELPA: 

(use-package elot
  :ensure t)

That is the whole installation. elot-mode is autoloaded.

2.1.3. Install ELOT from Git

Until the MELPA recipe lands, clone the repository and add it to your load path: 

git clone https://github.com/johanwk/elot ~/src/elot

In your init.el: 

(add-to-list 'load-path "~/src/elot/elot-package")
(require 'elot)

2.1.4. Point ELOT at ROBOT

Set elot-robot-jar-path to the absolute path of your downloaded robot.jar: 

(setq elot-robot-jar-path "~/bin/robot.jar")

You can also customize it via M-x customize-variable RET elot-robot-jar-path.

2.1.5. Verify

Open an Org file and run M-x elot-mode. The ELOT menu appears in the menu bar. M-x elot-version prints the loaded version.

If something goes wrong, see Troubleshooting and FAQ.

2.2. Your first ontology

In this chapter you build a tiny ontology of pets from scratch. By the end you will have:

  • a working pets.org file you can re-tangle and re-export at will,
  • a generated pets.omn ontology in OWL Manchester Syntax,
  • the start of an HTML render of your ontology document.

Time required: 15–20 minutes.

2.2.1. Create the file

Create pets.org anywhere in your filesystem and paste the skeleton below. Open it in Emacs and run M-x elot-mode. 

#+TITLE: A tiny ontology of pets
#+AUTHOR: Your Name

* Prefixes
,:PROPERTIES:
,:header-args:emacs-lisp+: :tangle no
,:END:

#+name: prefix-table
,| prefix | uri                                            |
,|--------+------------------------------------------------|
,| ex     | http://example.org/pets/                       |
,| owl    | http://www.w3.org/2002/07/owl#                 |
,| rdfs   | http://www.w3.org/2000/01/rdf-schema#          |
,| skos   | http://www.w3.org/2004/02/skos/core#           |

* ex:pets
,:PROPERTIES:
,:ID:       pets-ontology
,:ontology-iri: http://example.org/pets/
,:END:

A small example ontology used by the ELOT tutorial.

** Classes
*** ex:Animal
A living creature.

*** ex:Dog
,:PROPERTIES:
,:subclass-of: ex:Animal
,:END:
A dog is an animal.

*** ex:Cat
,:PROPERTIES:
,:subclass-of: ex:Animal
,:END:
A cat is an animal.

2.2.2. Tangle to OWL Manchester Syntax

With point anywhere in pets.org, run M-x elot-tangle. A file pets.omn appears next to your Org source. Open it; you should see Manchester-syntax declarations for ex:Animal, ex:Dog and ex:Cat.

2.2.3. Sanity-check with ROBOT

From a shell: 

robot report --input pets.omn --output pets-report.tsv

ROBOT's report should be empty (or contain only INFO entries). Errors mean ELOT generated something OWL does not like; see Linting and syntax checking.

2.2.4. Where to next

  • Adding axioms and annotations — give your classes proper definitions and relationships.
  • Exporting to OWL and HTML — produce a polished document.

2.5. A first SPARQL query and rdfpuml diagram

TODO — expand.

This chapter introduces two of ELOT's most-loved features:

  • running SPARQL queries directly against your tangled .omn via embedded source blocks,
  • generating PlantUML diagrams of selected triples via the rdfpuml-block Library-of-Babel helper.

2.5.1. A first SPARQL block

Add this to pets.org, below the class definitions: 

#+begin_src sparql :url "pets.omn"
,select ?s ?o where { ?s rdfs:subClassOf ?o }
#+end_src

Press C-c C-c inside the block. ELOT shells out to ROBOT and inserts the result table below the block. No PREFIX boilerplate needed — ELOT injects prefixes from your prefix-table.
